# text-to-tasks - プレーンテキストからタスク抽出・登録

貼り付けられたプレーンテキストを分析し、アクション可能なタスクを抽出して `my-task add` コマンドで登録する。

## 対象テキスト

あらゆる形式のプレーンテキストを対象とする:

- 議事録・会議メモ
- メール・チャットログ
- TODO リスト・箇条書きメモ
- Backlog / GitHub Issue の一覧コピー
- 自由記述のメモ・走り書き

## 実行手順

### 0. 環境確認

```bash
command -v my-task
```

コマンドが見つからない場合、ユーザーにエラーを報告し、以降の処理を一切行わない。

### 1. テキスト受け取り

`$ARGUMENTS` にテキストが含まれている場合はそれを使用する。
含まれていない場合、ユーザーにテキストの貼り付けを依頼する:

```
タスクを抽出したいテキストを貼り付けてください。
```

### 2. テキスト分析

貼り付けられたテキストから以下を抽出する:

**タスク候補の特定基準:**
- 動詞で始まる指示・依頼（「〜する」「〜を確認」「〜を修正」等）
- TODO / FIXME / アクションアイテム等のマーカー
- 担当者が割り当てられた作業項目
- 期限が言及された作業
- 箇条書きの各項目（タスクらしきもの）
- 「対応が必要」「要検討」等の行動を示唆する表現

**タスクらしくないものは除外する:**
- 単なる情報共有・報告（「〜が完了した」「〜の状況」等）
- 感想・コメント
- 前提条件の説明

**各タスク候補について以下を推測する:**

| フィールド | 推測方法 |
|-----------|---------|
| タイトル | テキストから簡潔なタスク名を生成（30文字以内目安） |
| プロジェクト | テキスト中のプロジェクト名、製品名、チーム名等から推測。判断できない場合は空欄 |
| 期限 | テキスト中の日付表現（「来週まで」「4/10」「金曜日」等）から推測。判断できない場合は空欄 |

**期限の変換ルール:**
- `my-task add --due` は `YYYY-MM-DD` または自然言語（`tomorrow`, `friday`, `来週` 等）を受け付ける
- テキスト中の日付表現を適切な形式に変換する
- 相対的な表現（「今週中」→ `friday`、「月末」→ その月末の `YYYY-MM-DD`）も変換する

### 3. 抽出結果の提示

抽出したタスク候補を一覧でユーザーに提示する:

```
【抽出されたタスク候補】

 #  タイトル                      プロジェクト   期限
 1  APIエンドポイントの追加        backend       2026-04-10
 2  ログイン画面のバグ修正         frontend      friday
 3  データベースマイグレーション    -             -

登録するタスクを選んでください:
  - 全て登録 → "all" または "全部"
  - 一部を選択 → 番号をカンマ区切り（例: "1,3"）
  - 修正したい → 番号と修正内容を指示（例: "2のプロジェクトをwebに変更"）
  - 中止 → "cancel" または "やめる"
```

### 4. ユーザー確認と修正

ユーザーの指示に応じて対応する:

- **全て登録**: 全タスクを登録に進む
- **番号指定**: 指定されたタスクのみ登録に進む
- **修正指示**: タスク内容を修正し、再度一覧を提示する（ステップ3に戻る）
- **中止**: 処理を停止する

### 5. タスク登録

確定したタスクを `my-task add` で1件ずつ登録する:

```bash
# プロジェクトと期限がある場合
my-task add "タスクタイトル" --project プロジェクト名 --due 期限

# プロジェクトのみの場合
my-task add "タスクタイトル" --project プロジェクト名

# 期限のみの場合
my-task add "タスクタイトル" --due 期限

# どちらもない場合
my-task add "タスクタイトル"
```

各登録後、`my-task add` の出力（`Added: #ID タイトル`）を記録する。

### 6. 完了報告

```
【タスク登録完了】

登録されたタスク:
  - #14 APIエンドポイントの追加 (project: backend, due: 2026-04-10)
  - #15 ログイン画面のバグ修正 (project: frontend, due: 2026-04-11)
  - #16 データベースマイグレーション

合計: 3件登録しました。
```

## 注意事項

- タスクタイトルは簡潔にする。元テキストが長文でも要約して30文字以内に収める
- プロジェクト名・期限が曖昧な場合は無理に推測せず空欄にする。誤った推測より空欄の方が良い
- 1つのテキストから大量のタスク（10件超）が抽出された場合、重要度の高いものから提示し、ユーザーに取捨選択を促す
- テキストが短すぎてタスクを抽出できない場合は、その旨を報告して追加情報を求める
